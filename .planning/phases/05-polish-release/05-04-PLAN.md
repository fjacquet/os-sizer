  <action>
Create `.github/workflows/deploy.yml` with this exact content:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-node@v6
        with:
          node-version: '22'
          cache: 'npm'
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v4
        with:
          path: dist
      - id: deployment
        uses: actions/deploy-pages@v5
```

Key differences from vcf-sizer:

- Branch is `main` (not `maincd`)
- Added `environment` block with `name: github-pages` and `url` output for deployment status tracking in GitHub UI
- Added `id: deployment` to the deploy step for URL output

This requires the repository to have GitHub Pages enabled with "GitHub Actions" as the source (Settings > Pages > Source: GitHub Actions). This is a user-side setting.
