## Milestone v2.1 Requirements

### Validation Fix (WARN)

- [x] **WARN-04**: User sees a live migration warning only when virtualization is enabled AND no RWX-capable storage (ODF or any other RWX storage class) is configured
- [x] **WARN-05**: Warning code, i18n keys, and test fixtures updated from `VIRT_RWX_REQUIRES_ODF` to `VIRT_RWX_STORAGE_REQUIRED` consistently across all 4 locales (EN/FR/DE/IT) and all test files

### PPTX Export (PPTX)

- [x] **PPTX-01**: User exports a consolidated single-slide PPTX containing title, KPI summary, chart, and BoM table on one slide
- [x] **PPTX-02**: Exported PPTX includes a vertical bar chart of node counts grouped by node pool type (using native pptxgenjs `BAR` chart API)
- [x] **PPTX-03**: Exported PPTX includes a stacked bar chart showing vCPU distribution per node pool when 3 or more distinct node pool types are present

### PDF Export (PDF)

- [x] **PDF-01**: Exported PDF includes a bar chart of node counts displayed above the BoM table
- [x] **PDF-02**: Exported PDF includes a KPI summary callout box showing total vCPU, total RAM, and total Storage between the chart and the BoM table
- [x] **PDF-03**: Exported PDF correctly renders accented and special characters for FR, DE, and IT locales (embedded Roboto Unicode font)
- [x] **PDF-04**: Exported PDF includes validation warnings rendered inline with severity-appropriate color and icon

### Multi-Cluster (CLUSTER)

- [x] **CLUSTER-01**: User can add, remove, and rename up to 5 independent clusters via a tab bar; each cluster has its own sizing configuration
- [x] **CLUSTER-02**: User can tag each cluster with a role: hub, spoke, or standalone
- [x] **CLUSTER-03**: User can view a side-by-side comparison table of all configured clusters (max 5 columns)
- [x] **CLUSTER-04**: PPTX, PDF, and CSV exports include per-cluster sections and an aggregate totals row

### Session Portability (SESSION)

- [x] **SESSION-01**: User can download the current sizing session as a `.json` file from the export toolbar
- [x] **SESSION-02**: User can load a previously saved `.json` session file; validated via Zod schema with user-friendly error messages
