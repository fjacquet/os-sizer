### ⚠ Tech Debt (2 of 17 flows)

**1. SNO_VIRT_NO_HA warning lost in store**
- `calcSNO()` in `calculators.ts` line 187 emits `SNO_VIRT_NO_HA` with `messageKey: 'warnings.sno.virtNoHa'`
- `calcCluster()` returns `{ sizing, warnings }` — the `warnings` array includes this warning
- `calculationStore.ts` line 17: `sizing: calcCluster(cluster).sizing` — `.warnings` is silently dropped
- Hardware minimum IS applied correctly (SNO_VIRT_MIN with 14/32/170 is enforced)
- Only the user-facing advisory message is never surfaced
- **Affected requirements:** SNO-01 (hardware correct, warning display missing)
- **Fix:** Extract and propagate `.warnings` in calculationStore, add `warnings.sno.virtNoHa` to all locale files

**2. Virt sub-inputs hidden for hcp/two-node-arbiter but calcVirt() still fires**
- `Step3ArchitectureForm.vue` line 187 gates virt sub-inputs on `standard-ha || compact-3node`
- `calcCluster()` has no topology guard — calls `calcVirt()` whenever `virtEnabled=true`
- For hcp/TNA users who happen to have virtEnabled=true, virt worker nodes appear in BoM with default vmCount=0 sizing
- **Fix:** Add topology guard in calcCluster() post-dispatch block, or add guard in calcVirt()
