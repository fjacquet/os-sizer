## Integration Findings

### Critical: Add-on Dispatch Missing

`calcODF` and `calcRHACM` from `src/engine/addons.ts` are exported via the engine barrel (`src/engine/index.ts`) but are **never called** from `src/engine/calculators.ts`. Every topology function hard-codes:

```typescript
odfNodes: null,
rhacmWorkers: null,
```

The fix is to call `calcODF(cluster.addOns.odfExtraOsdCount)` and `calcRHACM(cluster.addOns.rhacmManagedClusters)` when the respective add-on flags are enabled, and assign the results before returning from `calcCluster()` or from each topology function.

**Affected:** ENG-07, ENG-08, RES-05, FORM-05 (partial), RES-01 (partial)

### Significant: maxNodes UI Missing

`ClusterConfig.maxNodes` is used in `recommendation.ts` scoring logic (±10 to ±50 points depending on topology), but no UI input element is bound to this field in `Step1EnvironmentForm.vue`. The field permanently stays `null`, making all node-budget constraint scoring inactive.

**Affected:** REC-02, FORM-02 (partially)

### Minor: allocatableRamGB() Not Used in Production

`allocatableRamGB()` from `formulas.ts` (ENG-04) is exported, tested with tiered kernel reservation formula, but the `calcHCP()` function inlines `28.44` directly rather than calling it.
