## Interaction Contract

### Save Session Button

- **Position:** First among the two new session buttons, placed before "Share URL" in the toolbar flex row (rationale: save/load are session-management actions, grouped together at the start)
- **Label:** `t('results.toolbar.saveSession')` = "Save Session" (en)
- **State:** stateless — no loading spinner required (JSON serialization is synchronous; download triggers immediately)
- **Click behavior:** Calls `exportSession()`, triggers browser download of `os-sizer-session-YYYY-MM-DD.json`
- **Visual:** Same button class as Share/CSV/PDF/PPTX: `text-sm px-2 py-1 sm:px-3 sm:py-2 font-medium rounded border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-700 text-gray-700 dark:text-gray-200 hover:bg-gray-50 dark:hover:bg-gray-600 transition-colors`

### Load Session Button

- **Position:** Immediately after Save Session button in the toolbar flex row
- **Label:** `t('results.toolbar.loadSession')` = "Load Session" (en)
- **Click behavior:** Programmatically clicks the hidden `<input type="file">` element
- **Visual:** Same button class as Save Session
- **Disabled state:** Not applicable — button is always enabled

### Hidden File Input

- **Attributes:** `type="file"` `accept=".json"` `class="hidden"` `:key="fileInputKey"`
- **Key binding:** `fileInputKey` is a `ref<number>(0)`, incremented after every import attempt (success or failure) to force remount — ensures same file can be imported twice (D-08)
- **Change handler:** `handleLoadSession(event: Event)`

### Import Flow — Interaction States

| State | Visual | Duration |
|-------|--------|----------|
| Idle | No message below Load Session button | — |
| Success | Green text flash: `t('results.toolbar.sessionImportSuccess')` = "Session loaded successfully" below Load Session | 1500ms, then clears (mirrors `copied` ref pattern) |
| Error | Red inline text with ✖ prefix: `t('results.toolbar.sessionImportError')` below Load Session button | Persists until next import attempt |

**Error display markup pattern:**

```html
<p
  v-if="sessionImportError"
  class="text-sm text-red-700 dark:text-red-400 mt-1 flex items-center gap-1"
  role="alert"
>
  <span aria-hidden="true">✖</span>
  {{ sessionImportError }}
</p>
```

**Success flash markup pattern (mirrors `copied` ref):**

```html
<p
  v-if="sessionImportSuccess"
  class="text-sm text-green-600 dark:text-green-400 mt-1"
  role="status"
>
  {{ t('results.toolbar.sessionImportSuccess') }}
</p>
```

### Error Clearing Behavior

- `sessionImportError` clears immediately when the next import attempt begins (before FileReader resolves)
- `sessionImportSuccess` clears after 1500ms via `setTimeout` (same as `copied` ref in handleShare)
- Both `sessionImportError` and `sessionImportSuccess` are mutually exclusive — success clears error ref, error clears success ref

### Button Order in Toolbar

Final order (left to right): Save Session | Load Session | Share URL | Export CSV | Export PDF | Download PPTX

Rationale: session management actions grouped at the start, export/sharing actions following in the existing established sequence.

## Copywriting Contract

| Element | Copy (en) | i18n Key |
|---------|-----------|----------|
| Save Session button | "Save Session" | `results.toolbar.saveSession` |
| Load Session button | "Load Session" | `results.toolbar.loadSession` |
| Success flash | "Session loaded successfully" | `results.toolbar.sessionImportSuccess` |
| Import error (parse or schema failure) | "Invalid session file. Please select a valid os-sizer session." | `results.toolbar.sessionImportError` |
| Empty state heading | Not applicable — no empty state (toolbar always visible on results page) | — |
| Empty state body | Not applicable | — |

Destructive actions in this phase: **Load Session is a session-replacement action** — importing overwrites the current cluster configuration with no undo. Decision D-06 specifies no confirmation dialog; the replacement is immediate. This is intentional and within scope per CONTEXT.md. The success flash ("Session loaded successfully") serves as the confirmation signal.
